Pivot Analysis Tool

Note: I am not displaying original Mock-ups Or Screen shots due to NDA. However, I am trying to explain the design problem I solved in this project.

Pivot Analysis tool lets User to:
  • Ability to pivot (drill down) on a dimension value and continue analysis in different paths. (Feature that needed improvement)
  • And there are a few other features which were working fine.
Problem:

Whenever Users Drilled down on a Dimension, a new Analysis screen was added which made lead to multiple screens which became difficult after a certain number of drill downs were added. So, below is a simple workflow of how the tool works and how the drill down happens:

When drilled five steps deeper, page looked something like this, which lead to large amount of scrolling. Due to NDA, I am just showing depictive wireframes only.

Problems with this approach:
  • Too much scrolling,
  • Because each section is huge, if User is viewing a particular section, there are no visual cues which convey the level which is currently viewed.
Proposed Solution:
  1. Fixed Work Area: User should feel “in-control” and be able to manage most of the app related tasks from his main work area. Also, if a summary of every step of a drill down could be viewed at once, would make analysis easier.
    • Condensed Step + Expanded Step: To accommodate all (or most) steps of the drill down within the work area, all steps will be shown as thumbnails (condensed), but 1 step will be actual view (Expanded).
  2. Useful Visual Cues: More visual cues explaining the drill down at a simple glance. This helps reduce the cognitive workload for the Users.
    • Flow of Dimension Filters
    • Ability to change Dimension Filter quickly from Condensed View
